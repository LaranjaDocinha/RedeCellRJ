# Design Guidelines

Bem-vindo(a) às diretrizes de design do nosso projeto! Este documento serve como uma fonte central de verdade para todos os aspectos visuais e de interação, garantindo consistência e qualidade em toda a nossa aplicação.

## Espaçamento

Utilizamos um sistema de espaçamento baseado em uma escala definida por variáveis CSS em `index.css` para garantir layouts harmoniosos e responsivos. A "Base Unit" implícita parece ser `4px`.

- **xxs:** `4px`
- **xs:** `8px`
- **sm:** `16px`
- **md:** `24px`
- **lg:** `32px`
- **xl:** `48px`
- **xxl:** `64px`
- **xxxl:** `80px`
- **xxxxl:** `96px`

## Sombras (Shadows)

As sombras são definidas para criar profundidade e hierarquia visual, seguindo as variáveis CSS.
- **Elevation 1:** `0px 2px 1px -1px rgba(0,0,0,0.2), 0px 1px 1px 0px rgba(0,0,0,0.14), 0px 1px 3px 0px rgba(0,0,0,0.12)`
- **Elevation 2:** `0px 3px 3px -2px rgba(0,0,0,0.2), 0px 3px 4px 0px rgba(0,0,0,0.14), 0px 1px 8px 0px rgba(0,0,0,0.12)`

## Raio de Borda (Border Radius)

Definido para consistência nos cantos dos elementos.
- **Small:** `4px`
- **Medium:** `8px`
- **Large:** `12px`
- **Round:** `50%`

## Princípios de Material Design

Incorporamos os princípios do Material Design para criar uma experiência de usuário intuitiva e visualmente atraente.
- **Intenção e Significado:** O design deve ser proposital e reagir de forma significativa às interações do usuário.
- **Feedback Visual:** Fornecer feedback claro e imediato para todas as ações do usuário.
- **Hierarquia e Legibilidade:** O layout deve guiar o olho do usuário e apresentar informações de forma clara.
- **Responsividade:** A interface deve se adaptar a diferentes tamanhos de tela e dispositivos.
- **Acessibilidade:** Projetar para que todos os usuários, independentemente de suas capacidades, possam usar a aplicação.
