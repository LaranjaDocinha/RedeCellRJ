feat: Implementa melhorias de segurança, UI/UX e performance no fluxo de autenticação

Este commit introduz uma série de aprimoramentos no fluxo de login/registro, focando em segurança, experiência do usuário e otimização.

Backend:
- Adicionado Rate Limiting na rota de login para prevenir ataques de força bruta.
- Implementada prevenção de enumeração de usuários, padronizando mensagens de erro de login.
- Reforçada a política de senha forte para novos registros.
- Adicionado log de atividades para tentativas de login (sucesso e falha).

Frontend (UI/UX e Performance):
- Refatorado o componente de Login para utilizar o `api_helper` de forma consistente.
- Centralizado o feedback de sucesso e erro via `react-hot-toast` no `api_helper`.
- Implementado um novo layout assimétrico de duas colunas para a página de login.
- Adicionado background dinâmico com imagem de alta qualidade e overlay de gradiente.
- Aplicado efeito "Glassmorphism" ao painel do formulário de login.
- Incluída micro-interação de fade-in/scale-up no logo.
- Implementadas labels flutuantes e feedback de validação em tempo real com ícones.
- Adicionada animação de "tremor" no formulário em caso de erro de login.
- Configurado foco automático no campo de e-mail.
- Verificadas e mantidas as práticas de acessibilidade (HTML semântico, labels associadas, etc.).
- Adicionada pré-conexão com a API no `index.html` para otimização de performance.
- Confirmado o uso de code splitting via `React.lazy` para carregamento otimizado de páginas.
